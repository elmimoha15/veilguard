# Install on VS Code

Veilguard works in VS Code with MCP-compatible AI extensions such as **GitHub Copilot** (agent mode) or **Continue**.

---

## Step 1 — Add the MCP config

Open the Command Palette (`Ctrl+Shift+P` / `Cmd+Shift+P`), search **"Preferences: Open User Settings (JSON)"**, and add:

```json
"mcp": {
  "servers": {
    "veilguard": {
      "command": "npx",
      "args": ["-y", "--package=veilguard-cli", "veilguard-mcp"],
      "env": {
        "VEILGUARD_KEY": ""
      }
    }
  }
}
```

> **Alternative:** You can also add this to `.vscode/settings.json` in your project for project-level config.

---

## Step 2 — Add the rules file (recommended)

Download [`.cursorrules`](https://raw.githubusercontent.com/elmimoha15/veilguard/main/templates/cursorrules.txt) and save it to your project root.

VS Code AI agents read `.cursorrules` to know when to call each Veilguard tool. Without it, the tools still work but won't fire automatically.

---

## Step 3 — Restart VS Code

Close and reopen VS Code completely.

---

## Verify

Ask your AI agent: *"What Veilguard tools do you have access to?"*

It should list 14 security tools.

---

## Pro key

To activate Pro, add your license key to the MCP config:

```json
"env": {
  "VEILGUARD_KEY": "vg_live_xxxxxxxxxxxxxxxxxxxx"
}
```

Restart VS Code after saving. Get a key at [veilguard.dev/pro](https://veilguard.dev/pro).

---

## Notes

- MCP support in VS Code requires an agent extension with MCP compatibility (e.g. GitHub Copilot agent mode)
- The settings.json config is **user-level** (all projects) or **project-level** (`.vscode/settings.json`)
- `.cursorrules` is project-level — add it to each project you want protected
- Node.js 18 or later is required
