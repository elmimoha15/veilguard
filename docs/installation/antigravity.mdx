# Install on Antigravity

---

## Automatic setup (recommended)

Open your terminal, `cd` into your project root, and run:

```bash
npx veilguard-cli init
```

Veilguard detects the `.gemini/` folder and:
- Writes the MCP config to `.gemini/mcp.json`
- Writes `.antigravityrules` into your project root

Then **restart Antigravity**.

---

## Manual setup

### Step 1 тАФ Add the MCP config

Create `.gemini/mcp.json` in your project root:

```json
{
  "mcpServers": {
    "veilguard": {
      "command": "npx",
      "args": ["-y", "--package=veilguard-cli", "veilguard-mcp"],
      "env": {
        "VEILGUARD_KEY": ""
      }
    }
  }
}
```

Or use the Antigravity **MCP Settings Panel** to add the server:

| Field | Value |
|-------|-------|
| **Name** | `veilguard` |
| **Command** | `npx -y --package=veilguard-cli veilguard-mcp` |
| **Environment** | `VEILGUARD_KEY=` (empty for free) |

### Step 2 тАФ Add the security rules file

Create `.antigravityrules` in your project root. This file tells Antigravity's AI when to run security scans.

Generate it:
```bash
npx veilguard-cli init
```

Or copy the template from the [veilguard repository](https://github.com/elmimoha15/veilguard/blob/main/templates/antigravityrules.txt).

### Step 3 тАФ Restart Antigravity

Close and reopen Antigravity. The MCP server starts on the next launch.

Open `.gemini/mcp.json` and add your license key:

```json
{
  "mcpServers": {
    "veilguard": {
      "command": "npx",
      "args": ["-y", "--package=veilguard-cli", "veilguard-mcp"],
      "env": {
        "VEILGUARD_KEY": "vg_live_xxxxxxxxxxxxxxxxxxxx"
      }
    }
  }
}
```

Restart Antigravity after saving.

## How auto-scanning works in Antigravity

The `.antigravityrules` file instructs the AI agent to call Veilguard tools at specific moments:

- After you write or modify any file тЖТ `scan_secrets`
- After you create an API route тЖТ `scan_webhooks` + `scan_injection`
- After you change a Supabase migration or SQL file тЖТ `check_supabase_rls`
- After you modify `package.json` тЖТ `check_supply_chain` + `scan_dependencies`
- When you say "deploy", "go live", "ship it" тЖТ `full_audit`

Clean scans produce no output. The agent stays silent unless something is found.
