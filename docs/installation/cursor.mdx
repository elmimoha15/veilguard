# Install on Cursor

---

## Automatic setup (recommended)

Open your terminal, `cd` into your project root, and run:

```bash
npx veilguard-cli init
```

Veilguard detects the `.cursor/` folder and:
- Writes the MCP config to `.cursor/mcp.json`
- Writes `.cursorrules` into your project root

Then **restart Cursor**.

---

## Manual setup

### Step 1 â€” Add the MCP config

Cursor uses a **project-level** MCP config at `.cursor/mcp.json` inside your project folder.

Create the `.cursor/` directory if it doesn't exist, then create `.cursor/mcp.json`:

```json
{
  "mcpServers": {
    "veilguard": {
      "command": "npx",
      "args": ["-y", "--package=veilguard-cli", "veilguard-mcp"],
      "env": {
        "VEILGUARD_KEY": ""
      }
    }
  }
}
```

If the file already has other MCP servers, add the `veilguard` block inside `mcpServers`.

### Step 2 â€” Add the security rules file

Create `.cursorrules` in your project root. This file tells Cursor's AI agent when to run security scans automatically.

Generate it by running:
```bash
npx veilguard-cli init
```

Or copy the template from the [veilguard repository](https://github.com/elmimoha15/veilguard/blob/main/templates/cursorrules.txt).

### Step 3 â€” Restart Cursor

Close and reopen Cursor completely. The MCP server starts on the next launch.

Restart Cursor after saving.

## How auto-scanning works in Cursor

The `.cursorrules` file instructs Cursor's AI agent to call Veilguard tools at specific moments:

- After you write or modify any file â†’ `scan_secrets`
- After you create an API route â†’ `scan_webhooks` + `scan_injection`
- After you change a Supabase migration or SQL file â†’ `check_supabase_rls`
- After you modify `package.json` â†’ `check_supply_chain` + `scan_dependencies`
- When you say "deploy", "go live", "ship it" â†’ `full_audit`

Clean scans produce no output. The agent stays silent unless something is found.

- The `.cursor/mcp.json` config is **project-level** â€” you need to set it up in each project

- `.cursorrules` is project-level â€” run `init` in each project you want protected
