# Install on Windsurf

---

## Automatic setup (recommended)

Open your terminal, `cd` into your project root, and run:

```bash
npx veilguard-cli init
```

Veilguard detects the `~/.windsurf/` folder and:
- Writes the MCP config to `~/.windsurf/mcp.json`
- Writes `.windsurfrules` into your project root

Then **restart Windsurf**.

---

## Manual setup

### Step 1 â€” Add the MCP config

Windsurf uses a **global** MCP config at `~/.windsurf/mcp.json` (applies to all projects).

Create or edit that file:

```json
{
  "mcpServers": {
    "veilguard": {
      "command": "npx",
      "args": ["-y", "--package=veilguard-cli", "veilguard-mcp"],
      "env": {
        "VEILGUARD_KEY": ""
      }
    }
  }
}
```

If the file already exists with other MCP servers, add the `veilguard` block inside `mcpServers`.

### Step 2 â€” Add the security rules file

Create `.windsurfrules` in your project root. This file tells Cascade (Windsurf's AI) when to run security scans.

You can generate the file content by running:
```bash
npx veilguard-cli init
```

Or copy the rules template manually from the [veilguard repository](https://github.com/elmimoha15/veilguard/blob/main/templates/windsurfrules.txt).

### Step 3 â€” Restart Windsurf

Close and reopen Windsurf completely. The MCP server starts on the next launch.

---

## Verify the installation

After restarting, check that Veilguard is active:

1. Open the **Tools** panel in Windsurf
2. Go to **MCP** section
3. **Veilguard** should appear with a green status indicator

Or ask Cascade: *"What Veilguard tools do you have access to?"* â€” it should list all 14 tools.

If you have a Pro license, add your key to `~/.windsurf/mcp.json`:

```json
{
  "mcpServers": {
    "veilguard": {
      "command": "npx",
      "args": ["-y", "--package=veilguard-cli", "veilguard-mcp"],
      "env": {
        "VEILGUARD_KEY": "vg_live_xxxxxxxxxxxxxxxxxxxx"
      }
    }
  }

```

Restart Windsurf after saving.

## How auto-scanning works in Windsurf

The `.windsurfrules` file instructs Cascade to call Veilguard tools at specific moments:

- After you write or modify any file â†’ `scan_secrets`
- After you create an API route â†’ `scan_webhooks` + `scan_injection`
- After you change a Supabase migration or SQL file â†’ `check_supabase_rls`
- After you modify `package.json` â†’ `check_supply_chain` + `scan_dependencies`
- When you say "deploy", "go live", "ship it" â†’ `full_audit`

Clean scans produce no output. Cascade stays silent unless something is found.

## Notes

- The `~/.windsurf/mcp.json` config is **global** â€” it applies to every project you open in Windsurf
- The `.windsurfrules` file is **project-level** â€” run `init` in each project you want protected
- Node.js 18 or later is required
