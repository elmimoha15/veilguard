# Full Audit

The `full_audit` tool runs all 13 codebase scanners at once (every scanner except Security Headers, which needs a deployed URL), calculates a security score, assigns a letter grade (A+ to F), and generates an AI-ready fix prompt you can paste into your agent to fix everything at once.

**Pro only.** Pro gets **unlimited** full audits with the grade and AI-ready fix prompt.  
**Free does not get the full audit at all** — calling it returns an upgrade prompt. (Free users still run the 14 individual scanners for unlimited vulnerability alerts.)

---

## What it scans

`full_audit` runs every Veilguard scanner in one pass:

1. `scan_secrets` — hardcoded API keys
2. `check_env` — environment variable exposure
3. `check_git` — git history and gitignore gaps
4. `check_auth_config` — authentication misconfigurations
5. `scan_webhooks` — unsigned webhook handlers
6. `scan_injection` — SQL/NoSQL/command injection, IDOR, mass assignment
7. `check_cors` — CORS misconfiguration
8. `check_supply_chain` — malicious packages
9. `scan_dependencies` — known CVEs
10. `check_supabase_rls` — Row Level Security policies
11. `check_firebase` — Firebase security rules
12. `scan_app_security` — rate limiting, password storage, file uploads, error exposure, sensitive logging, open redirects

---

## Scoring system

The audit starts at **100 points** and deducts points per finding based on real-world impact:

| Finding type | Points deducted |
|-------------|----------------|
| Hardcoded secret | −20 per finding |
| Unsigned webhook | −20 per finding |
| SQL/NoSQL/command injection | −20 per finding |
| Missing Supabase RLS / open Firebase rules | −20 per finding |
| IDOR (Insecure Direct Object Reference) | −20 per finding |
| Mass assignment | −15 per finding |
| Missing rate limiting on auth/payment routes | −15 per finding |
| CORS wildcard | −10 per finding |
| Sensitive data in logs | −10 per finding |
| Stack trace / error object exposed to client | −8 per finding |
| Missing security headers | −5 per finding |
| All other criticals (fallback) | −15 per finding |
| All other warnings (fallback) | −5 per finding |
| Info findings | −1 per finding |

Score floor is **0**. Maximum is **100**.

## Grade table

| Grade | Score | Plain English meaning |
|-------|-------|-----------------------|
| **A+** | 95–100 | Your app is production-ready. Excellent security hygiene. |
| **A** | 90–94 | Your app is production-ready. Excellent security hygiene. |
| **B+** | 85–89 | Good security. A few improvements recommended before scaling. |
| **B** | 80–84 | Good security. A few improvements recommended before scaling. |
| **C+** | 75–79 | Moderate risk. Fix warnings before you get real users. |
| **C** | 70–74 | Moderate risk. Fix warnings before you get real users. |
| **D** | 60–69 | High risk. Critical issues present. Fix before deploying. |
| **F** | 0–59 | Do NOT deploy. Your app has critical vulnerabilities that attackers actively exploit. |

## Report format (Pro)

A full audit report has five sections:

### 1. Header
```
🛡️ Veilguard Security Audit
─────────────────────────────
Grade: B+  |  Score: 87/100  |  Found: 2 issues
```

### 2. Critical findings (if any)
```
─────────────────────────────
🔴 CRITICAL — Fix before deploying (1 issue)
─────────────────────────────
1. Anyone can fake a payment and get your product for free
   → File: src/app/api/webhooks/stripe/route.ts:14
   → Fix: Wrap your handler with Stripe's constructEvent() to verify
          the webhook signature before processing any event.
```

### 3. Warnings (if any)
```
─────────────────────────────
🟡 WARNINGS — Fix soon (1 issue)
─────────────────────────────
2. Your app has no browser protection — users are vulnerable to clickjacking
   → Fix: Add Content-Security-Policy and X-Frame-Options headers in
          next.config.js or middleware.ts
```

### 4. Grade meaning
```
─────────────────────────────
📊 What this grade means
─────────────────────────────
Good security. A few improvements recommended before scaling.
```

### 5. AI Fix Prompt
```
─────────────────────────────
🔧 AI Fix Prompt
─────────────────────────────
Copy and paste this into your AI agent to fix everything at once:

Fix the following security issues in my project:

1. [CRITICAL] Webhook handler missing signature verification
   File: src/app/api/webhooks/stripe/route.ts:14
   Fix: Wrap your handler with Stripe's constructEvent()...

Apply all fixes while maintaining existing functionality.
```

### 6. What passed
```
─────────────────────────────
✅ What passed (11 checks)
─────────────────────────────
• No hardcoded API keys or secrets found in your source code
• Environment variables are properly configured
• Git history is clean
• Authentication setup looks solid
• No SQL injection or unsafe input found
• CORS is configured safely
• No malicious or typosquatted packages detected
• No dependency vulnerabilities found
• Supabase Row Level Security policies look safe
• Firebase security rules look safe
• App-layer checks passed (rate limiting, IDOR, uploads, logging, etc.)
```

```

The full security audit grades your whole project (A+ to F) and writes an
AI-ready fix prompt that patches every issue at once. It's unlimited on Pro.

Want your grade and the one-paste fix? Upgrade to Pro:
→ veilguard.dev/pro ($19/mo or $149/yr — save 35%)

Meanwhile, the individual scanners still run free and will alert you to any
vulnerability they find (the fixes for those are unlocked with Pro too).

Add your key to the MCP config: VEILGUARD_KEY=your_key_here
```

## Access

The full audit is **Pro-only and unlimited**. **Free does not get any full audit.**

The 14 individual scanners run with no limit on either tier — free sees the alerts, Pro sees the fixes.

## Triggering a full audit

Your AI calls `full_audit` automatically when you use any phrase implying a full security check. You can also ask directly:

- *"Run a full audit on my project"*
- *"What's my security grade?"*
- *"Is my app safe to deploy?"*
- *"Audit my code"*

See [Auto-Scanning](/docs/auto-scanning) for the full list of trigger phrases.
